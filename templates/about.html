About

The idea is to have an architecture built around CouchDB. All the accessions and metadata related to accessions, is saved in this central couch instance. This allows for easy replication and well defined API.

How is data retrieved?

I’d like to avoid using a monolithic system. Since data upload is well defined through the CouchDB API, we can benefit from different methods:

  • DDC – or a system equally powerful for letting people upload data
  • Web Scraping
  • Staging Area
  • etc..

All these systems will talk directly with a couch. The cool thing is that the Couch can live anywhere – it can exist in someones local gene bank network and then it can be replicated to the other couch instances all over the internet. This can become a truly decentralized architecture for accessions.

Frontends

With data storage out the way we can now build all kinds of interfaces for this data. Couchapps or completely standalone apps written in any language – all they need to do is communicate with the couch API.

This could give burst to a whole new ecosystem of applications that take advantage of this data. For example:

  • Statistical apps
  • Map apps with with cooperation with ArcGIS
  • Search apps. Elastic Search already works great together with couch’s api.
